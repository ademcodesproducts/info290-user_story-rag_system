TITLE: Multi-Workspace Fleet Administration Console

PERSONA:
Role: Platform Admin / Data Platform Lead
Company size: 2,000–50,000 employees
Context: Manages 10–30 Databricks workspaces across multiple cloud regions and business units. Currently logs into each workspace individually for every administrative task. Spends ~3 hours/week on cross-workspace cost reporting alone.
Job-to-be-done: Administer, monitor, and govern a fleet of Databricks workspaces from a single interface — without writing and maintaining custom API scripts.

PAIN POINTS:
1. "When finance asks why spend is up 40% this week, I log into 18 workspaces, export dashboards from each, and assemble it manually. Three hours, every time." — Platform Admin, $2.1M ACV enterprise customer (TKT-015).
2. Cluster policy updates must be applied to each workspace individually — no broadcast or template propagation. An admin managing 20 workspaces makes 20 separate edits for each policy change.
3. No cross-workspace job run history — determining which workspace had a job failure last Tuesday requires checking each workspace's job run log.
4. The Account Console (accounts.cloud.databricks.com) shows billing summary and metastore assignment but no workspace-level resource visibility.
5. Unity Catalog lineage stops at workspace boundaries — cross-workspace data flows are invisible to the lineage graph.

REQUIREMENTS:
1. Fleet cost dashboard in Account Console: aggregated and per-workspace spend for any date range, broken down by SKU and (where tagged) by team. No custom SQL required.
2. Cross-workspace job run feed: a unified view of all job runs across all workspaces in the account, filterable by status (failed/running/succeeded), workspace, and date range.
3. Centralized cluster policy management: admins define policies in the Account Console and push them to selected workspaces (push-to-all or multi-select). Policy version history tracked per workspace.
4. Cross-workspace user inventory: single view of all users across all workspaces, their workspace assignments, and last active timestamp. Supports bulk deprovisioning.
5. Alert routing: admins configure account-level alerts (cost anomaly, job failure rate, cluster policy violation) that aggregate signals across all workspaces.

SUCCESS CRITERIA:
- A platform admin produces a cross-workspace cost report for any 30-day window in under 5 minutes from the Account Console with no API calls or custom scripts.
- A cluster policy change is propagated to all selected workspaces in a single operation in under 60 seconds.
- Cross-workspace job run feed loads the last 24 hours of runs across 20 workspaces in under 10 seconds.

OPEN QUESTIONS:
- How do we handle workspaces where the account admin does not have workspace-admin privileges?
- Should cross-workspace lineage be part of this PRD or a separate Unity Catalog initiative?
