PAIN POINTS RAISED:

1. Andile: "We promoted a model to production this sprint using the new Unity Catalog Model Registry. The promotion took one API call and there was no approval gate, no review, no audit record. For a fraud detection model that influences financial decisions, that's a governance problem. Our compliance team is asking us how we demonstrate oversight. I don't have an answer."

2. Fatima: "UC migration completed this sprint — final 400 tables migrated. But we lost lineage history for about 15% of the tables. The migration tool didn't transfer lineage edges from the old metastore. We now have tables in Unity Catalog with no upstream lineage and we have to reconstruct it manually by reading pipeline code."

3. Jordan: "Analysts are now using the Unity Catalog explorer regularly which is great. But the search is weak. You can search by table name but not by column name or by description. If an analyst wants to find 'which table has a column about customer lifetime value,' they can't. They ask me instead, and I've become a human catalog search engine."

DECISIONS:
- Andile to document the manual approval process (Slack → confirmation → promote) as a formal SOP while waiting for native approval workflows.
- Fatima to file a bug with Databricks on lineage migration gaps — include a specific list of affected table types with reproducible examples.

OPEN QUESTION:
Does Unity Catalog support full-text search on column descriptions and tags? If not, is it on the roadmap? Fatima to check the roadmap session from Data + AI Summit recordings.
