PAIN POINTS RAISED:

1. Lily: "We still haven't figured out why our event counts were off last quarter. I'm now pretty sure it was the watermark threshold silently dropping late-arriving events. I added explicit logging in the DLT pipeline to count late events — and sure enough, we're dropping about 2% of events that arrive more than 4 hours late. The DLT docs on watermarking are sparse. I pieced this together from Stack Overflow."

2. Rohit: "We need to update the gold table transformation logic. In a normal world, I'd just update the code and run a targeted refresh. In DLT world, that means an 8-hour full pipeline refresh. We've been putting this change off for 3 sprints because nobody wants to own the 8-hour refresh window."

3. Isabel: "Auto Loader schema evolution bit us again. An upstream team added a column to their Kafka output. Auto Loader picked it up fine. But they also changed an existing column from string to JSON — and Auto Loader failed the stream with an incompatible schema error. We were down for 2 hours while I diagnosed it."

DECISIONS:
- Lily to write internal documentation on DLT watermarking behavior and late data handling — the tribal knowledge should be written down.
- Team agrees to split the problematic pipeline into two smaller ones before the gold table logic update, to reduce the refresh scope. Tomás to add the split work to the Q3 backlog.

OPEN QUESTION:
Is there a way to do a "schema-only refresh" in DLT that updates the pipeline definition without reprocessing data? Rohit to research and report back.
