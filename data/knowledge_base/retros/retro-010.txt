PAIN POINTS RAISED:

1. Carlos: "We're paying for MLflow artifact storage that's completely useless. I queried the experiment list this week — we have 847 experiments, most of them abandoned. Artifacts going back 18 months that nobody will ever look at. There's no way to bulk clean them up from the UI. I started writing a cleanup script but it's going to take a week to run through the API rate limits."

2. Omar: "I found a data leakage issue in our churn model this sprint — the same look-ahead bias problem that the insurance team reported. Our Feature Store point-in-time join was using the wrong timestamp column. Six weeks in production before we caught it, same story. We caught it earlier this time only because I happened to plot the feature timestamp distribution as a sanity check. There should be a built-in check for this."

3. Andile: "Model explainability for our credit scoring model is becoming urgent — regulators are asking for decision explanations. We're using SHAP but it's all custom code outside MLflow. The SHAP values aren't logged anywhere auditable. If we get an audit, I have to hope I can reconstruct the exact model version and SHAP computation from Git history."

DECISIONS:
- Carlos to open a feature request for MLflow artifact retention policies — attach a script that demonstrates the API-rate-limited cleanup problem as evidence of why native tooling is needed.
- Omar to add temporal validation checks as standard pre-training assertions in the team's model training template notebook.

OPEN QUESTION:
Should we standardize on a model card format and log it as an MLflow artifact for every model that enters staging? Would help with both explainability and audit trail. Yuna to draft a template.
