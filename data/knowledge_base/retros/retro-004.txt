PAIN POINTS RAISED:

1. Sandra: "Finance asked me again this week for a team-level cost breakdown. I spent 3 hours assembling it from billing exports and tag data. Same 3 hours I spend every week. We have 14 workspaces. I cannot keep doing this manually."

2. Ben: "The SCIM sync failed twice this sprint. Both times we had users who should have been deprovisioned still showing as active in Databricks for 4–6 hours. Kenji's team flagged it as a security issue. The SCIM endpoint returns a 500 with 'internal server error' — there's nothing I can do to fix it on my end."

3. Amara: "We pushed a cluster policy update to 14 workspaces this sprint. Fourteen. Separate. Logins. I updated the same policy field 14 times. If I missed one — which I have done before — we have inconsistent governance across workspaces with no way to detect it."

DECISIONS:
- Ben to implement a compensating control: a Lambda function that checks Okta deactivations every 15 minutes and directly calls the Databricks SCIM API to deactivate the user. Not a fix, but reduces the exposure window from 6 hours to 15 minutes.
- Sandra to document the cost reporting process as a runbook so it can be delegated. Tag it as a candidate for automation once Databricks ships a proper cost attribution feature.

OPEN QUESTION:
Is Terraform the right tool for managing cluster policies across 14 workspaces consistently? Amara to prototype a Terraform module and demo next sprint.
