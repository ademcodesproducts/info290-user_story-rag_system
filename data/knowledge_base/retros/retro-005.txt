PAIN POINTS RAISED:

2. Omar: "My morning setup ritual — start cluster, install packages, reload data, redefine functions — is 25 minutes every day. That's over 2 hours per week of setup that produces zero value. I've started keeping a cluster alive overnight just to avoid it, which I know is wasteful."

3. Nadia: "The notebook merge conflict this sprint was bad. Yuki and I were both working on the feature engineering notebook. We resolved it eventually but the JSON diff was completely unreadable. We ended up manually reconstructing both changes from memory because comparing the JSON was harder than just rewriting."

DECISIONS:
- Ravi to request platform admin set personal cluster idle termination to 4 hours instead of 1 hour — reduces the overnight cost trade-off.
- Team to add PAT expiry date to team wiki and set a shared calendar reminder 7 days before expiry. Workaround for missing token expiry warnings in the extension.

Should we evaluate Databricks Serverless compute for interactive work? Removes cluster startup latency entirely but Ravi is unsure about cost implications for our workload profile. Action: compare 1-week cost of current setup vs. serverless.
