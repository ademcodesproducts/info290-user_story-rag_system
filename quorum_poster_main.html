§ 01 Motivation
"PMs live across Jira, Confluence, Slack, and Notion — yet no tool synthesises this knowledge into a coherent voice for decision-making."
Observed pattern · Databricks PM interviews

Product managers synthesise heterogeneous artefacts — customer interviews, PRDs, retros, and support tickets — but existing RAG systems treat all documents as interchangeable text chunks, destroying structural semantics.

Quorum introduces document-type-aware retrieval with agentic multi-step query planning, producing structured PM outputs rather than raw passages.

§ 02 Problem Statement
P1 Structural information loss — flat chunking collapses speaker turns in interviews and ignores section hierarchy in PRDs.
P2 Vocabulary mismatch — BM25-only retrieval misses semantic synonyms; dense-only misses exact product/version terms.
P3 Single-hop limitation — one retrieval pass cannot handle multi-faceted PM queries spanning personas, pain points, and roadmap items.
P4 No structured output — existing tools return passages, not actionable PM artefacts (user stories, gap flags, roadmap items).

§ 05 Two-Tier Index Design
Tier 1
Document Summary Index
One embedding per document. Encodes document-level metadata, type tags, and abstractive summary. Used for coarse candidate shortlisting across 100 documents before fine retrieval.
Tier 2
Chunk Detail Index
Type-aware segments: speaker turns for interviews, sections for PRDs/retros, issue blocks for tickets. Fine-grained passage-level matching with metadata filtering.
§ 06 Domain Objects & Structured Output

Quorum generates typed PM artefacts rather than free-form passages, enabling downstream consumption by roadmap tools and audit workflows.

Persona RoadmapItem SpeakerTurn Citation GapFlag

Gap detection uses DeBERTa-v3-large NLI to surface unmet needs that appear across multiple customer signals but have no corresponding roadmap item — a core PM workflow Quorum uniquely enables.

§ 10 Future Work
1. Live workspace integration — ingest real Jira tickets, Confluence pages, and Slack threads via API connectors with incremental indexing.
2. Domain-tuned embeddings — fine-tune on PM vocabulary to improve pain keyword recall beyond the current 19.6% baseline.
3. Cross-project synthesis — federated retrieval across multiple product areas and workspaces for org-wide PM intelligence.
4. Human-in-the-loop validation — structured PM review step to validate GapFlags and RoadmapItems before export to planning tools.
